# SoulBuyer

Продвинутая система скупщика ресурсов для Paper **1.21+**: живое GUI, динамический рынок, ротация ассортимента, **магазин бустеров**, **автопродажа с тремя режимами**, **интеграция с PlayerPoints** (отдельный донатный скупщик или выплата целиком в поинты), прогрессия игрока и несколько режимов хранения — от YAML на одном сервере до MySQL + Redis в сети.

---

## Описание

**SoulBuyer** — это не «положи предмет в слот и жди». Игрок открывает меню и видит **сетку скупаемых ресурсов** с актуальными ценами, очками и рыночным коэффициентом. ЛКМ по предмету — **продажа всего такого ресурса из инвентаря**; ПКМ — **выбор количества** в отдельном подменю. Есть фильтры по категориям, сортировки, пагинация и кнопка «продать всё».

Чем больше на сервере продают один ресурс — тем ниже его **рыночный коэффициент** (с настраиваемым минимумом и постепенным восстановлением). За продажи начисляются **очки прогрессии** и опыт **категорий** (руды, мобы, растения, блоки, прочее); очки можно тратить в **магазине бустеров** на временные усиления. Выплата — через **Vault** или **PlayerPoints** (см. [PlayerPoints и донатный скупщик](#playerpoints-и-донатный-скупщик)).

**Автопродажа** — отдельный донат-плюс: игрок сам выбирает, *когда* сдавать ресурсы — при подборе, при открытии меню скупщика или **содержимое сундука при открытии**. Настраиваются категории, уведомления и минимальная цена.

Ассортимент не застаивается: по таймеру меняется набор скупаемых предметов, сбрасываются рыночные коэффициенты, игроки получают оповещение в чат. Внизу меню — **живая панель статистики**: сколько осталось до обновления, что продано сегодня, сколько ресурсов сейчас в ротации.

## Особенности

Кратко, зачем ставить на сервер:

- **Из коробки** — ~380 предметов, GUI, ротация, рынок и lang (ru/en) с первого запуска; без ручной возни с конфигами
- **Всё настраивается** — цены, каталог, слоты GUI, права, лимиты рынка, ротация, автопродажа, economy; тексты в `lang/`, цифры в `config.yml` / `items.yml`
- **GUI вместо команд** — продажа, фильтры, сортировки, количество, автопродажа; команды только как алиасы
- **Живой рынок** — цена падает от массовых сдач, decay возвращает к норме; игрок видит коэффициент в lore
- **Ротация ассортимента** — по таймеру новый набор из пула; оповещение в чат; открытые меню обновляются на месте
- **Донатный скупщик** — Vault + PlayerPoints: два меню (`/buyer` и `/donbuyer`) или один скупщик целиком на поинты; два флага в config
- **Автопродажа** — донат-плюс: подбор / меню / сундук; категории, мин. цена, уведомления; при dual — выбор, куда платить (монеты или поинты)
- **Бустеры** — магазин временных усилений в меню скупщика; оплата очками прогрессии, Vault или PlayerPoints (настраивается)
- **Персональные лимиты продаж** — дневной cap на item-id (анти-демпинг); права LuckPerms и бустер лимитов
- **Прогрессия** — очки, XP категорий, VIP-множители через LuckPerms; мотивация фармить, не только лутать
- **Живая статистика** — таймер до ротации, «продано сегодня», размер ассортимента прямо в меню (+ PlaceholderAPI)
- **Анимация категорий** — иконки фильтров крутят реальные предметы из текущей ротации
- **Оптимизация** — весь I/O БД асинхронно; main thread не блокируется; HikariCP; Redis только когда нужна сеть
- **Хранение на выбор** — `flat` / `sqlite` / `mysql` + Redis для нескольких Paper-серверов с общим рынком
- **Безопасные продажи** — secure storage на транзакцию; при ошибке или выходе предметы возвращаются
- **API для других плагинов** — `SoulBuyerApi`: меню, котировки, продажа из своего GUI, категории предметов, прогресс

## GUI скупщика

| Зона | Что делает |
|------|------------|
| **Верхний ряд** | Фильтры категорий + блок «Ваш прогресс» (очки, множитель) |
| **Сетка 21 слот** | Иконки скупаемых предметов: цена, очки, рынок, количество в инвентаре. **ЛКМ** — продать всё, **ПКМ** — подменю количества |
| **Стрелки ← →** | Пагинация (на 4-м ряду, по бокам от сетки) |
| **Нижний ряд** | Сортировка (одна кнопка-переключатель), «продать всё», **бустеры**, **автопродажа**, **панель «До обновления»** |
| **Категории** | «Все» — иконка Vault (хранилище); остальные — **живая анимация**: иконка меняется и показывает реальный предмет из текущего ассортимента категории |

**Сортировка** (комparator, клик — следующий режим): дороже → дешевле → больше очков → меньше очков.

**Бустеры** (звезда незера в нижнем ряду): **ЛКМ** — магазин временных усилений. Доступно из **обычного** и **донатного** меню. Подробности — в разделе [Бустеры](#бустеры).

**Автопродажа** (изумруд в нижнем ряду): **ЛКМ** — экран настроек, **ПКМ** — вкл/выкл. Подробности — в разделе [Автопродажа](#автопродажа).

**Тексты и цвета** — в `lang/ru.yml`, `lang/en.yml`. Hex-цвета, градиентный заголовок, lore у категорий и предметов настраиваются без пересборки JAR.

### Иконки в меню (шаблоны кузнеца, отделки брони)

В `gui/buyer.yml` → `hideVanillaItemTooltip: true` (дефолт): в tooltip остаётся только lore SoulBuyer — без ванильных строк «Smithing Template», «Applies to», атрибутов и т.п.

| Тип предмета | Как рисуется в слоте |
|--------------|----------------------|
| Обычные ресурсы | Иконка предмета + ваш lore |
| **Отделки брони** (bolt, flow, coast…) | Железный нагрудник **с узором отделки** |
| Netherite Upgrade | Слиток незерита |

**Как проверить после обновления**

1. Собери JAR, замени на сервере, перезапусти сервер.
2. Открой `/buyer` → категория «Прочее» (или где сейчас отделки в ротации).
3. В слоте должна быть **отделка на нагруднике**, а не голый серый нагрудник без рисунка.
4. Наведи курсор — только название и цена SoulBuyer, без ванильного мусора.
5. То же в автопродаже: изумруд → **ПКМ** по категории → отделки с узором.

**Если иконка всё ещё не та**

- В `plugins/SoulBuyer/gui/buyer.yml` должно быть `hideVanillaItemTooltip: true`.
- Клиент **1.21+**; выключи **F3+H** (расширенные подсказки путают).
- Админ: в `config.yml` → `debug-tooltip: true` → `/soulbuyer admin reload` → `/soulbuyer admin debug-tooltip` → `/buyer`. На проде держи **false** (дефолт).

## Автопродажа

Отдельная механика для доната (`soulbuyer.autosell`, по умолчанию OP). Игрок настраивает поведение сам — настройки сохраняются per-player (YAML или MySQL).

### Когда продавать

Один из трёх режимов (переключается кнопкой в меню настроек):

| Режим | Иконка | Когда срабатывает |
|-------|--------|-------------------|
| **При подборе** | Воронка | Предмет попал в инвентарь с земли или с рыбалки |
| **При открытии меню** | Изумруд | Открыт GUI скупщика — продаёт всё подходящее **из инвентаря игрока** |
| **При открытии сундука** | Сундук | Открыт контейнер — продаёт всё подходящее **из слотов сундука**. Инвентарь игрока не трогается |

Работает с сундуком, бочкой, шалкером, эндер-сундуком, воронкой, раздатчиком, выбрасывателем, вагонеткой-сундуком. Учитываются **включённые категории**, **выбор предметов**, **мин. цена** и **текущая ротация** ассортимента (продаются только item-id, которые сейчас в скупщике).

### Что ещё настраивается

| Опция | Смысл |
|-------|--------|
| **Вкл/выкл** | Мастер-переключатель (центр меню или ПКМ по изумруду в скупщике) |
| **Категории** | Руды, мобы, растения, блоки, прочее — **ЛКМ** вкл/выкл категорию целиком |
| **Предметы в категории** | **ПКМ** по иконке категории → экран выбора: какие item-id из этой категории автопродавать |
| **Уведомления** | Action bar / чат / выкл |
| **Мин. цена** | Не продавать предметы дешевле порога (с учётом рынка и множителей) |
| **Куда продавать** | Только при dual buyer: обычный скупщик (Vault) или донатный (PlayerPoints). Клик по слитку в центре 4-го ряда |

### Выбор предметов в категории

После включения категории по умолчанию автопродаются **все предметы этой категории из текущей ротации**. Чтобы оставить, например, только кости из «Мобов» или только алмазы из «Руд»:

1. Меню автопродажи → **ПКМ** по нужной категории.
2. Откроется сетка **только активных** item-id (как в `/buyer` сейчас).
3. **Клик** по предмету — вкл/выкл автопродажу для него (подсветка = включён).
4. Нельзя выключить **последний** включённый предмет в активной категории.

Если предмет **выпал из ротации** — он не показывается в выборе и **не автопродаётся**, даже если раньше был включён. После возврата в ротацию настройка игрока сохраняется (если id не в списке отключённых).

**Пример:** категория «Мобы» включена → ПКМ → выключить всё лишнее, оставить только `bone` → при подборе/сундуке/меню продаются только кости из мобов (и только пока кости в ротации).

### Хранение настроек игрока

Per-player в `data/autosell/{uuid}.yml` (flat) или в колонке `settings` таблицы `soulbuyer_autosell` (MySQL). Текстовый формат:

```
enabled=true
trigger=pickup
notify=actionbar
min-unit-price=0
payout=vault
categories=ores,mobs,plants,blocks,misc
disabled-items=bone,rotten_flesh
```

| Ключ | Смысл |
|------|--------|
| `categories` | Включённые категории (через запятую) |
| `disabled-items` | Item-id, **исключённые** из автопродажи внутри включённых категорий. Пусто = все активные предметы категории |

Остальные ключи — режим триггера, уведомления, мин. цена, выплата.

На слоте с книгой — краткая справка для игрока. Если в инвентаре нечего продавать, лишних сообщений в чат нет.

В **обоих** меню скупщика (обычном и `/donbuyer`) кнопка автопродажи показывает текущий режим выплаты; настройка — в том же экране автопродажи, куда игрок зашёл из любого меню.

### Зачем это серверу

- **Донат-плюс** без pay-to-win: те же цены и рынок, меньше рутины.
- **Сортировка базы**: режим «сундук» — открыл хранилище, скупаемый мусор внутри сразу продался, нужное осталось в сундуке.
- **АФК-фарм**: режим «подбор» — подобрал ресурс, он уже продан.
- **Казуал-игроки**: режим «меню» — зашёл в `/buyer`, всё лишнее сдалось само.

В `config.yml` → секция `autosell`: вкл/выкл фичи на сервере, дефолты для новых игроков, шаги мин. цены, задержка после подбора.

## Бустеры

Магазин **временных усилений** — открывается из обычного (`/buyer`) и донатного (`/donbuyer`) меню: кнопка **«Бустеры»** (звезда незера) → отдельный экран с тремя предложениями.

Покупка списывает валюту из `config.yml` → `boosters.currency`:

| Значение | Списание |
|----------|----------|
| `progression_points` | **Очки SoulBuyer** (дефолт — зарабатываются за продажи, раньше некуда тратить) |
| `vault` | Монеты через Vault |
| `playerpoints` | PlayerPoints |

### Три бустера (дефолты, всё в конфиге)

| ID | Эффект | Длительность | Цена (дефолт) |
|----|--------|--------------|---------------|
| `multiplier` | **+0.5** к личному множителю дохода | 1 ч | 125 очков |
| `money` | **×1.25** к монетам/поинтам от продажи | 1 ч | 175 очков |
| `limit` | **×2** к персональному дневному лимиту на item-id | 30 мин | 75 очков |

Цены, длительность, сила эффекта, материалы и слоты в GUI — в `config.yml` → `boosters.offers` и `gui/boosters.yml`.

На кнопке каждого бустера в lore: эффект, цена, длительность, **активен ли сейчас** и сколько осталось. Повторная покупка **обновляет** таймер того же типа.

### Лимиты продаж (связано с бустером `limit`)

Параллельная механика **анти-демпинга** — секция `sell-limits` в `config.yml`:

- У каждого игрока **дневной лимит** на продажу каждого `item-id` (дефолт 64 шт./сутки).
- LuckPerms-ноды `soulbuyer.limit.vip` / `soulbuyer.limit.premium` поднимают cap (128 / 256).
- Бустер лимитов умножает итоговый лимит.
- `sell-limits.enabled: false` — лимиты полностью отключены.

При превышении лимита продажа **урезается** до допустимого объёма, лишнее возвращается в инвентарь.

### Права и fresh install

| Нода | Доступ |
|------|--------|
| `soulbuyer.boosters` | Магазин бустеров (default: true) |

Дефолты в Java: бустеры **включены**, валюта — **очки прогрессии**, все три предложения и GUI на месте после первого запуска. Активные бустеры сохраняются per-player (`data/boosters/` или таблица `soulbuyer_boosters`).

## PlayerPoints и донатный скупщик

SoulBuyer **не привязан только к Vault**. Если на сервере стоит [PlayerPoints](https://www.spigotmc.org/resources/playerpoints.80745/), плагин умеет платить за сдачу ресурсов **донатными поинтами** — без костылей, отдельными командами и с тем же рынком, ротацией и прогрессией.

Один каталог `items.yml`, одни цены и коэффициенты — меняется только **валюта выплаты** (монеты сервера или PlayerPoints).

### Три режима (секция `economy` в `config.yml`)

| `player-points-enabled` | `donate-buyer-enabled` | Что видит игрок |
|---------------------------|------------------------|-----------------|
| `false` | — | Классика: `/buyer` → выплата в **Vault** |
| `true` | `false` | **Один** скупщик: `/buyer` → всё в **PlayerPoints** (Vault не нужен) |
| `true` | `true` | **Два** скупщика: обычный за монеты + **донатный** за поинты |

```yaml
economy:
  player-points-enabled: false   # включить PlayerPoints
  donate-buyer-enabled: false    # true = второе меню только за поинты
```

### Два скупщика — зачем это круто

На донат-серверах часто нужно разделить экономику: **фарм → монеты**, **донатный контент → поинты**. SoulBuyer закрывает оба сценария одним JAR:

| Меню | Команда | Выплата | Кому |
|------|---------|---------|------|
| **Обычный** | `/buyer`, `/sell`, `/soulbuyer` | Vault (монеты) | Все с `soulbuyer.use` |
| **Донатный** | `/donbuyer`, `/dbuyer`, `/ppbuyer` | PlayerPoints | С `soulbuyer.donate` |

- Тот же ассортимент и **живой рынок** — цена падает от массовых продаж в обоих меню.
- **Очки прогрессии** и XP категорий начисляются как обычно (это не валюта, а мета-прогресс SoulBuyer).
- Отдельный **заголовок и lore** у донатного меню — игрок сразу видит, что платят поинтами.
- **Автопродажа** следует выбору игрока в настройках: при dual-режиме — Vault или PlayerPoints; если включён только PlayerPoints — всегда в поинты.

### Fresh install

Дефолты в Java: PlayerPoints **выключен**, работает только Vault — как раньше. Включение — две строки в `config.yml`, перезапуск сервера. Команды `/donbuyer` и алиасы уже в `plugin.yml`; если донатный режим выключен, `/donbuyer` ответит, что фича недоступна.

### Зависимости

| Плагин | Когда нужен |
|--------|-------------|
| **Vault** + экономика | Дефолт; обязателен при dual-режиме и классическом режиме |
| **PlayerPoints** | `player-points-enabled: true` |

Оба — `softdepend` в `plugin.yml`: плагин стартует, ждёт нужную экономику и только потом открывает GUI.

## Динамический рынок

- Каждый ресурс имеет **базовую цену** и **коэффициент рынка** (старт 1.0).
- Массовые продажи **снижают** коэффициент до настроенного минимума.
- **Decay** постепенно возвращает цену к 1.0, если ресурс перестают сдавать.
- В режиме **mysql + сеть** коэффициенты синхронизируются через Redis pub/sub.

Игрок видит множитель рынка прямо в lore предмета в GUI.

## Ротация ассортимента

Полный каталог (~380 ванильных ресурсов в `items.yml`) — это **пул**. В скупщике одновременно активен **случайный поднабор** (по умолчанию 48 предметов, минимум из каждой категории).

| Параметр | Дефолт | Смысл |
|----------|--------|-------|
| `catalog-rotation.enabled` | `true` | Ротация вкл/выкл |
| `interval-seconds` | `3600` | Интервал смены (1 час) |
| `active-item-count` | `48` | Сколько предметов в меню |
| `reset-market-on-rotation` | `true` | Сброс коэффициентов рынка при смене |
| `notify.enabled` | `true` | Сообщение в чат при обновлении |

Состояние ротации хранится в `data/rotation.yml`. Открытые меню **обновляются без закрытия**.

## Панель статистики (центр низа)

Вместо «Страница 1/3» — **часы с живым таймером** (обновление каждую секунду):

- **До обновления** — оставшееся время до ротации
- **Сегодня продано** — монеты, очки, позиций
- **В ассортименте** — сколько ресурсов сейчас в ротации

Плейсхолдеры для lore в lang:

| Плейсхолдер | Описание |
|-------------|----------|
| `{rotation_left}` | Таймер до ротации или «отключена» / «скоро» |
| `{sold_today_money}` | Монет за сегодня |
| `{sold_today_points}` | Очков за сегодня |
| `{sold_today_stacks}` | Позиций продано |
| `{active_items}` | Предметов в текущем ассортименте |

## Прогрессия

- **Очки** за каждую продажу (из `base-points` предмета + бонус от базовой цены × `points-per-currency`).
- **XP категорий** — растёт от заработанных очков; бонус к **деньгам** считается от **уровней** XP (не от сырого числа), с потолком `max-category-bonus`.
- **Permission-множители** — VIP, Premium и др. (берётся максимальный из выданных).
- **Бонус доминирующей категории** — доп. доход от уровня XP главной категории (`category-xp-per-level`, `dominant-category-bonus-per-level`).
- Очки можно **тратить** в магазине бустеров (если `boosters.currency: progression_points`).
- Защита от разгона: `max-unit-price`, `max-payout-per-sale` на одну транзакцию.

---

## Защита и надёжность

- **Secure storage** на время транзакции — предметы не теряются при лаге или ошибке.
- При выходе / сбое — **возврат** непроданного в инвентарь.
- Продажи пишутся в **лог** (`sales.log` или таблица `soulbuyer_sales`) — основа для статистики «сегодня продано».
- Весь I/O с БД — **асинхронно**, main thread не блокируется.

## Команды

Основная: `/soulbuyer`. Алиасы открытия меню (настраиваются): `/buyer`, `/sell`, `/sb`, `/rbuyer`, `/bs`.

**Донатный скупщик** (только при `economy.donate-buyer-enabled: true`): `/donbuyer`. Алиасы: `/dbuyer`, `/ppbuyer`, `/donatesell`.

| Команда | Действие |
|---------|----------|
| `/soulbuyer` | Открыть GUI скупщика (Vault или PlayerPoints — по режиму `economy`) |
| `/buyer`, `/sell`, … | То же (алиасы) |
| `/donbuyer`, `/dbuyer`, … | Донатное меню — выплата в **PlayerPoints** (dual-режим) |
| `/soulbuyer admin reload` | Перезагрузить config, gui, lang |

> Алиасы наследуют подкоманды: `/buyer admin reload` = `/soulbuyer admin reload`.


## Публичное API

Интерфейс `SoulBuyerApi` регистрируется в Bukkit `ServicesManager`. Другой плагин подключается через `depend` или `softdepend: [SoulBuyer]`.

### Подключение

```java
import bm.b0b0b0.soulBuyer.api.SoulBuyerApi;
import bm.b0b0b0.soulBuyer.api.SoulBuyerApiProvider;
import bm.b0b0b0.soulBuyer.api.SoulBuyerSellDelivery;
import bm.b0b0b0.soulBuyer.api.SoulBuyerSellReturnPolicy;
import org.bukkit.Bukkit;

SoulBuyerApi api = SoulBuyerApiProvider.get();
// или: Bukkit.getServicesManager().load(SoulBuyerApi.class)

if (!api.isReady()) {
    return;
}
```

`SoulBuyerApiProvider.get()` всегда возвращает объект: если SoulBuyer ещё не готов — stub с `isReady() == false` и no-op методами.

### Справочник методов

| Метод | Возврат | Описание |
|-------|---------|----------|
| `isReady()` | `boolean` | Плагин загружен, каталог и экономика готовы |
| `openBuyerMenu(Player)` | — | Открыть стандартное GUI скупщика |
| `quoteItem(Player, itemId)` | `Optional<ItemUnitQuote>` | Цена и очки за 1 шт. с учётом рынка, множителей и кол-ва в инвентаре |
| `quoteStack(Player, ItemStack)` | `Optional<ItemUnitQuote>` | То же по предмету в руке / слоте |
| `marketCoefficient(itemId)` | `double` | Текущий коэффициент рынка (1.0 = база) |
| `cachedPoints(Player)` | `double` | Очки прогрессии из кэша |
| `cachedMultiplier(Player)` | `double` | Итоговый личный множитель (permission × категория + бустеры) |
| `cachedCategoryXp(Player, categoryId)` | `double` | XP категории из кэша |
| `fetchProgress(UUID)` | `CompletableFuture<PlayerProgress>` | Async загрузка прогресса из хранилища; обновляет кэш |
| `isSellable(itemId)` | `boolean` | Предмет есть в пуле `items.yml` |
| `isInActiveCatalog(itemId)` | `boolean` | Предмет в текущей ротации ассортимента |
| `activeCatalogSize()` | `int` | Сколько предметов сейчас в ротации |
| `categoryId(itemId)` | `Optional<String>` | Категория из `items.yml` (`ores`, `mobs`, `plants`, `blocks`, `misc`, …) |
| `categoryId(ItemStack)` | `Optional<String>` | Категория по material + custom model data |
| `isAutosellFeatureEnabled()` | `boolean` | Автопродажа включена на сервере |
| `canUseAutosell(Player)` | `boolean` | У игрока право на автопродажу |
| `isAutosellEnabled(Player)` | `boolean` | Мастер-переключатель автопродажи у игрока (не детали категорий/предметов) |
| `isSaleInProgress(Player)` | `boolean` | Идёт транзакция продажи (secure storage занят) |
| `sellAll(Player)` | `boolean` | Продать всё скупаемое из инвентаря |
| `sellAll(Player, Runnable)` | `boolean` | То же + callback по завершении |
| `sellAll(Player, SoulBuyerSellDelivery, Runnable)` | `boolean` | Продажа с выбором канала уведомления |
| `sellItem(Player, itemId)` | `boolean` | Продать весь item-id из инвентаря |
| `sellItem(Player, itemId, Runnable)` | `boolean` | То же + callback |
| `sellItemAmount(Player, itemId, amount, Runnable)` | `boolean` | Продать указанное количество из инвентаря |
| `sellStacks(Player, stacks)` | `boolean` | Продать переданные стеки (см. ниже) |
| `sellStacks(Player, stacks, Runnable)` | `boolean` | То же + callback |
| `sellStacks(Player, stacks, returnPolicy, delivery, Runnable)` | `boolean` | Полный контроль политики возврата и уведомлений |

Методы продажи возвращают `false`, если API не готов, игрок null, список пуст или уже идёт другая продажа. `true` — транзакция **принята**; результат смотри в callback и сообщениях игроку.

### Модели

**`ItemUnitQuote`** — `unitPrice`, `unitPoints`, `marketCoefficient`, `playerMultiplier`, `inventoryAmount`.

**`PlayerProgress`** — `playerId`, `points`, `categoryXp` (map categoryId → XP).

### Продажа из инвентаря

`sellAll`, `sellItem`, `sellItemAmount` работают как кнопки в GUI SoulBuyer: забирают предметы из инвентаря игрока, применяют лимиты, бустеры, рынок, прогрессию, выплату (Vault / PlayerPoints по конфигу). При ошибке непроданное возвращается в инвентарь.

```java
if (api.isSaleInProgress(player)) {
    return;
}
api.sellAll(player, SoulBuyerSellDelivery.ACTION_BAR, () -> {
    // main thread, после завершения транзакции
    player.sendMessage("Готово!");
});
```

### Продажа из своего GUI — `sellStacks`

Для кастомного меню с кнопкой «Продать всё»:

1. Собери `List<ItemStack>` из слотов своего GUI.
2. Отфильтруй по категории через `categoryId`, если нужно.
3. **Убери стеки из GUI** до вызова (при `CALLER_OWNS_ITEMS`).
4. Вызови `sellStacks`.

SoulBuyer сам считает цену, лимиты, бустеры, очки, XP категорий и платит игроку. Несколько стеков в одном вызове — одна транзакция.

```java
List<ItemStack> toSell = new ArrayList<>();
for (ItemStack stack : guiSlots) {
    if (stack == null || stack.isEmpty()) {
        continue;
    }
    api.categoryId(stack).filter("ores"::equals).ifPresent(category -> toSell.add(stack.clone()));
}
if (toSell.isEmpty() || api.isSaleInProgress(player)) {
    return;
}

// убрать toSell из своего GUI до вызова
boolean started = api.sellStacks(
        player,
        toSell,
        SoulBuyerSellReturnPolicy.CALLER_OWNS_ITEMS,
        SoulBuyerSellDelivery.SILENT,
        () -> refreshMyGui(player)
);
if (!started) {
    // вернуть предметы в GUI вручную
}
```

| `SoulBuyerSellReturnPolicy` | Поведение при отмене / ошибке |
|-----------------------------|-------------------------------|
| `RETURN_TO_PLAYER` | Дефолт. Непроданное возвращается в инвентарь игрока |
| `CALLER_OWNS_ITEMS` | SoulBuyer не трогает предметы — откат GUI делает ваш плагин |

| `SoulBuyerSellDelivery` | Уведомление игроку |
|-------------------------|-------------------|
| `CHAT` | Сообщение в чат (дефолт) |
| `ACTION_BAR` | Action bar |
| `SILENT` | Без сообщений SoulBuyer |

Часть стеков может не продаться из‑за **дневного лимита** — лишнее вернётся игроку (или останется у caller при `CALLER_OWNS_ITEMS`).

### Категории — `categoryId`

Категория берётся из поля `category` в `items.yml` для каждого item-id. Типичные значения: `ores`, `mobs`, `plants`, `blocks`, `misc`. Свои id допустимы — главное, чтобы совпадали с фильтрами в GUI и автопродаже.

```java
api.categoryId("raw_iron");           // Optional["ores"]
api.categoryId(player.getInventory().getItemInMainHand()); // по material + CMD
api.isSellable("raw_iron");          // true, если id в пуле
api.isInActiveCatalog("raw_iron");   // true, если сейчас в ротации
```

### Кэш прогресса

`cachedPoints` / `cachedMultiplier` / `cachedCategoryXp` читают память. После join данные подгружаются async; для точных значений до открытия меню вызови `fetchProgress(uuid).join()` или дождись callback.

`fetchProgress` безопасен с async-потока; мутация Bukkit API из его callback — только через `runTask`.

**Автопродажа и API:** выбор категорий и отдельных item-id настраивается только через GUI SoulBuyer (`gui/autosell.yml`). Методов `toggleAutosellItem` / `getDisabledItems` в `SoulBuyerApi` нет — для кастомных меню используй `sellStacks` и свою фильтрацию по `categoryId` / `isInActiveCatalog`.
